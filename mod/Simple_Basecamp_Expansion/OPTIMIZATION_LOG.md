## 🚀 v1.1.0 - EoC系统优化（2026-06-01）

### 优化概述

本次优化主要针对Effect on Condition (EoC) 系统进行性能优化和规范修正。

### 1. 性能优化

#### 1.1 合并EVENT EoC（性能提升87.5%）

**问题**：
- 原有8个独立的EVENT EoC监听同一事件
- 每次获得物品都会触发8次检查
- 造成不必要的性能开销

**优化**：
- 合并为1个统一的EVENT EoC
- 使用`or`条件检查所有研究报告
- 使用`if/then`分支处理不同研究

**效果**：
- ✅ 触发次数：8次 → 1次
- ✅ 性能提升：87.5%
- ✅ 代码更简洁

**代码示例**：
```json
{
  "type": "effect_on_condition",
  "id": "EOC_RESEARCH_COMPLETED",
  "eoc_type": "EVENT",
  "required_event": "character_gains_item",
  "global": false,
  "condition": {
    "or": [
      // 8个研究报告的检查
    ]
  },
  "effect": [
    // 使用if/then分支处理
  ]
}
```

#### 1.2 优化里程碑检查（性能提升0-66%）

**问题**：
- 无条件调用3个里程碑EoC
- 即使不满足条件也会执行

**优化**：
- 使用`if/then`条件判断
- 只在满足条件时才调用

**效果**：
- ✅ 减少0-66%的不必要调用
- ✅ 更精确的触发逻辑

#### 1.3 优化RECURRING频率（性能提升50%）

**问题**：
- 进度提醒频率过高（6-12小时）

**优化**：
- 调整为12-24小时
- 添加deactivate_condition

**效果**：
- ✅ 减少50%的检查频率
- ✅ 自动停用机制

### 2. 规范修正

#### 2.1 变量类型修正

**问题**：
- 数值变量使用字符串
- 布尔变量使用"yes"/"no"

**修正**：
```json
// Before
{ "u_set_var": "research_count", "value": "0" }

// After
{ "u_set_var": "research_count", "value": 0 }
```

**符合官方文档要求** ✅

#### 2.2 布尔检查优化

**问题**：
- 使用值比较检查布尔状态

**优化**：
- 使用变量存在性检查

```json
// Before
{ "u_has_var": "milestone_first_research", "value": "no" }

// After
{ "not": { "u_has_var": "milestone_first_research" } }
```

#### 2.3 添加global标记

**问题**：
- EVENT类型EoC未明确指定global

**修正**：
```json
"eoc_type": "EVENT",
"global": false  // 明确指定
```

### 3. 代码质量提升

#### 3.1 添加详细注释

```json
"//": "=== 优化后的研究追踪系统 ===",
"//": "性能优化：将8个独立EVENT EoC合并为1个，减少87.5%的触发次数",
"//": "规范优化：使用正确的变量类型（整数和布尔）"
```

#### 3.2 简化变量初始化

**Before**：15个变量初始化  
**After**：3个变量初始化（其他使用变量存在性）

**减少80%** ✅

## 🔍 技术细节

### EoC类型选择

| 类型 | 使用场景 | 性能影响 |
|------|---------|---------|
| EVENT | 事件触发 | 低 |
| RECURRING | 定期检查 | 中 |
| ACTIVATION | 手动调用 | 低 |
| SCENARIO_SPECIFIC | 场景初始化 | 极低 |

### 变量系统设计

```
全局变量（优化后）：
├── basecamp_expansion_initialized (标记)
├── research_count (整数 0-8)
├── story_chapter (整数 0-5)
├── research_xxx_done (存在性标记) x8
├── milestone_xxx (存在性标记) x3
└── welcome_shown (存在性标记)
```

### 条件优化策略

1. **使用or合并相似条件**
2. **使用if/then分支处理**
3. **使用变量存在性代替值比较**
4. **使用deactivate_condition自动停用**
